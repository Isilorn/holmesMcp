# ADR-0009 — Réutilisation des scripts jeedom-audit

- **Date** : 2026-05-03 (esquisse) / TBD (finalisation J0)
- **Statut** : draft
- **Source brief** : §7 (Relation aux scripts Python existants), D7.1-D7.4

## Contexte

7 scripts Python existent dans jeedom-audit : `db_query.py`, `logs_query.py`, `api_call.py`, `usage_graph.py`, `scenario_tree_walker.py`, `resolve_cmd_refs.py`, `setup.py`. Ces scripts encapsulent une logique métier Jeedom (parsing, graphe d'usage, résolution `#[O][E][C]#`, escape SQL réservés) éprouvée par l'usage. Holmes MCP peut en réutiliser la logique sans en copier les couches d'accès (SSH → local, HTTP distant → localhost).

La posture clean room s'applique aussi à jeedom-audit : la réutilisation est autorisée (c'est le code du PO) mais les couches d'accès externes sont remplacées par des accès locaux.

## Décision pressentie

**Réutilisation par modules de logique** (Option D du brief) : la logique métier est adaptée, pas copiée textellement. Les couches d'accès sont remplacées.

Mapping :

| Script jeedom-audit | Module Holmes MCP | Traitement |
|---|---|---|
| `db_query.py` | `_core/db.py` | Logique escape conservée, couche SSH→driver Python local |
| `logs_query.py` | `_core/logs.py` | Validation/résolution chemins conservées, couche SSH→lecture fichier local |
| `api_call.py` | `_core/api.py` | Quasi intact, IP localhost au lieu de distante, blacklist renforcée D4bis.5 |
| `usage_graph.py` | `_domain/usage_graph.py` | Logique conservée, `db.py` au lieu de SSH |
| `scenario_tree_walker.py` | `_domain/scenario_walker.py` | Idem |
| `resolve_cmd_refs.py` | `_domain/cmd_refs.py` | Idem |
| `setup.py` | Jeté | Remplacé par hooks plugin + config UI Jeedom |
| `_common/sensitive_fields.py` | `_domain/sanitize.py` | Logique étendue (3 mécanismes, mask + count) |

**Référence permanente** : chaque module `_core/` et `_domain/` référence en commentaire d'en-tête le script jeedom-audit dont il dérive (nom, commit/tag précis — D7.4).

## Conséquences pressenties

- Cohérence de logique entre jeedom-audit et Holmes MCP (même escape SQL, même parsing)
- Divergences futures documentées dans les commentaires d'en-tête
- `setup.py` jeté : aucun setup côté client, tout dans les hooks plugin Jeedom
- La matrice D5.8 (ADR-0019 future) validera que la logique portée couvre tous les WF jeedom-audit
