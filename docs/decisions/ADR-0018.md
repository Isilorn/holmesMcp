# ADR-0018 — Résultat POC D2.3 : faisabilité daemon Python sur Bookworm

- **Date** : 2026-05-03 (J0-2 + J0-3)
- **Statut** : accepted
- **Source brief** : D2.3 (POC faisabilité daemon Python sur Bookworm — 7 hypothèses)

## Contexte

Le brief identifiait 7 hypothèses à valider avant de s'engager sur l'architecture daemon Python + Streamable HTTP :

1. Python 3.11+ disponible sur Bookworm sans action manuelle
2. SDK MCP officiel installable dans un venv Jeedom natif
3. FastMCP démarre et écoute sur le port configuré
4. `tools/list` retourne les tools enregistrés via le protocole MCP
5. Le daemon survive au redémarrage Jeedom (lancé/arrêté via `deamon_start/stop`)
6. PID file correct, `deamon_info()` retourne `state: ok`
7. Claude Desktop peut se connecter au daemon depuis le LAN (HTTP non-TLS)

L'hypothèse #7 était identifiée comme le principal risque (🔴 Élevé dans PROJECT_STATE). Le plan B prévu était HTTPS self-signed généré automatiquement à l'install.

## Résultat des hypothèses (J0-2 — 2026-05-03)

| # | Hypothèse | Résultat |
|---|---|---|
| 1 | Python 3.11+ sur Bookworm | ✅ Python 3.11.2 présent |
| 2 | SDK MCP installable en venv natif Jeedom | ✅ mcp==1.27.0 installé via packages.json |
| 3 | FastMCP démarre sur le port configuré | ✅ Uvicorn sur 0.0.0.0:8765 |
| 4 | `tools/list` retourne tool `hello` | ✅ Validé via curl depuis devbox |

| 7 | Claude Desktop HTTP non-TLS LAN | ❌ Refusé — Claude Desktop rejette les URLs `http://` |

## Décision

**Plan B activé : HTTPS self-signed généré automatiquement à l'install.**

Claude Desktop refuse systématiquement les configurations MCP avec une URL `http://`. Le rejet intervient au parsing du `claude_desktop_config.json` — le serveur n'est même pas contacté. L'URL doit être `https://` pour être acceptée comme configuration valide.

Le mécanisme retenu :
- Génération d'un certificat self-signed à l'install (`openssl req -x509 ...`)
- Certificat et clé stockés dans `/etc/holmes_mcp/` (chmod 640, root:www-data)
- Le daemon expose HTTPS via uvicorn avec `ssl_keyfile` + `ssl_certfile`
- L'utilisateur accepte le certificat une fois dans son client MCP (ou le confie à son store)
- L'URL d'accès devient `https://<ip-jeedom>:8765/mcp`

## Conséquences

- `post_install.sh` : ajouter la génération du certificat self-signed (`openssl`)
- `holmesMcpd.py` : passer `ssl_keyfile` et `ssl_certfile` à uvicorn
- `holmesMcp.class.php` : `getMcpUrl()` retourne `https://` au lieu de `http://`
- Page config UI : afficher l'URL HTTPS + note sur l'acceptation du certificat
- Documentation : expliquer la démarche d'acceptation certificat dans Claude Desktop / Cursor
- Coût estimé : +1 session Claude Code (J0-3bis), à absorber dans J0
