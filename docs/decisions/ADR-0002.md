# ADR-0002 — Stack technologique

- **Date** : 2026-05-03 (esquisse) / 2026-05-03 (finalisation J0-1)
- **Statut** : proposed
- **Source brief** : §9 (Stack technologique), D2.1, D9.1, D9.2, D9.3, D9.4

## Contexte

Le daemon Python requiert un SDK MCP officiel, un driver MySQL, des bibliothèques HTTP, de parsing SQL et de logging. Ces choix impactent la maintenabilité, la facilité de diagnostic pour un PO non-dev, et la friction d'installation sur Bookworm. Les bibliothèques tierces doivent être déclarables dans `plugin_info/packages.json` (méthode moderne Jeedom 4.4.9+).

Les détails précis (versions épinglées, choix entre alternatives) sont délégués à Claude Code en J0 (décisions 🔵 D1.2, D4bis.1, D9.1, D11.6) après vérification empirique des compatibilités.

## Décision — J0-1 (D9.1 + D11.6 arrêtés)

| Couche | Choix **arrêté** | Rationale |
| --- | --- | --- |
| Daemon | Python 3.11+ | SDK MCP Python officiel Anthropic en premier servi |
| SDK MCP | `mcp` (PyPI, officiel Anthropic) | Alignement spec, support Streamable HTTP |
| Transport MCP | Streamable HTTP (spec 2025-03-26+) | Seul transport viable pour accès réseau |
| Driver MySQL | PyMySQL — **à confirmer J0-2** (D4bis.1 🔵) | Pure Python, compatible MariaDB 10.x |
| HTTP localhost | **`httpx`** ✅ D9.1 | SDK MCP async (anyio) → httpx s'intègre nativement |
| Parsing SQL | **`sqlparse`** ✅ D9.1 | Éprouvé, PyPI top 1000, gère SELECT-only validation |
| Logging | **`structlog`** ✅ D9.1 | JSON Lines D14.1, structuré, compatible asyncio |
| Tests | `pytest` + `pytest-asyncio` | Standard Python, asyncio_mode=auto |
| Lint/format | **`ruff`** ✅ D11.6 | Remplace flake8+black, unifié, très rapide en CI |
| Gestion deps | `packages.json` (apt + pip3) + `system::getCmdPython3()` | Méthode moderne Jeedom 4.4.9+ |

**Décisions 🔵 tranchées en J0-1** : D9.1 (httpx, sqlparse, structlog), D11.6 (ruff).
**Décision 🔵 reportée à J0-2** : D4bis.1 (PyMySQL — test CREATE USER sur box requis).

**Absent de la stack V1** : pas de framework web tiers (SDK MCP embarque son serveur HTTP), pas d'ORM (SQL direct), pas de table DB interne (stockage via mécanismes Jeedom natifs).

## Conséquences

- `packages.json` déclare les deps apt + pip3 ; `requirements.txt` est une référence documentation uniquement
- Entrypoint daemon utilise stdlib `logging` pour le bootstrap (avant import structlog) ; structlog intégré en J1
- Toute dep supplémentaire = ADR ou modification de cette ADR (traçabilité)
