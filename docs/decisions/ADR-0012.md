# ADR-0012 — Stratégie de tests

- **Date** : 2026-05-03 (esquisse) / TBD (finalisation J0)
- **Statut** : draft
- **Source brief** : §11 (Tests), D11.1-D11.6

## Contexte

Les tests doivent couvrir trois niveaux : unitaire (logique pure, sans dépendances), intégration (stack DB réelle mais sur fixtures synthétiques), live (sur la box réelle du PO). La box réelle n'est pas accessible depuis la CI publique, mais Claude Code y a accès via SSH (alias `Jeedom`). Les évals LLM-side (qualité de réponse) sont hors scope V1.

## Décision pressentie

**Pyramide de tests V1** :

| Niveau | Localisation | Fixtures | Exécution | CI |
|---|---|---|---|---|
| Unitaires | `tests/unit/` | Synthétiques + mocks | Claude Code | Oui (GitHub Actions) |
| Intégration | `tests/integration/` | MySQL Docker (fixtures synthétiques) | Claude Code | Oui (GitHub Actions) |
| Live | `tests/live/` (checklists) | Réelles (hors repo, workspace local CC) | Claude Code via SSH | Non (pas de box en CI) |

**Couvertures cibles** :
- `_core/` et `_domain/` : >80%
- `_domain/sanitize.py` : **100% obligatoire**

**Deux jeux de fixtures** :
- **Synthétiques** : générées par Claude Code, versionnées dans `tests/fixtures/synthetic/`, utilisées en CI
- **Réelles** : récupérées via SSH par Claude Code, sanitisées, hors repo (workspace local uniquement)

**Aucune action opérationnelle PO** pour les tests unitaires et d'intégration. Validation Claude Desktop (D8.3 critère #4) = test ponctuel par release, conduit par le PO.

**CI GitHub Actions minimale** (`ci.yml`) : lint `ruff` + tests unit + tests intégration sur container Docker MySQL, déclenchés sur push `develop` et PR vers `main`.

## Conséquences pressenties

- La CI publique n'a pas accès SSH, pas de credentials box PO, pas de dépendances réelles
- Les tests live (SSH Claude Code) valident les cas que les fixtures synthétiques ne peuvent pas couvrir
- Le niveau 100% sur `sanitize.py` est non négociable : c'est le filet de sécurité principal contre les fuites de credentials
