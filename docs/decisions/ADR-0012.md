# ADR-0012 — Stratégie de tests

- **Date** : 2026-05-03 (esquisse) / 2026-05-04 (finalisation J1-3)
- **Statut** : accepted
- **Source brief** : §11 (Tests), D11.1-D11.6

## Contexte

Les tests doivent couvrir trois niveaux : unitaire (logique pure, sans dépendances), intégration (stack réelle), live (sur la box réelle du PO). La box réelle n'est pas accessible depuis la CI publique, mais Claude Code y a accès via SSH (alias `Jeedom`). Les évals LLM-side (qualité de réponse) sont hors scope V1.

ADR-0018 (résultat POC D2.3) a établi que Claude Code exécute les tests live sur la box via SSH — les tests d'intégration font partie de ce périmètre.

## Décision

**Pyramide de tests V1** :

| Niveau | Localisation | Fixtures | Exécution | CI |
|---|---|---|---|---|
| Unitaires | `tests/unit/` | Synthétiques + mocks | Claude Code | Oui (GitHub Actions) |
| Intégration | `tests/integration/` | Box réelle via SSH | Claude Code | **Non** — skip auto si absent |
| Live | `tests/live/` (checklists) | Réelles (hors repo, workspace local CC) | Claude Code via SSH | Non |

**Pivot par rapport au plan initial** : les tests d'intégration passent sur la **box réelle** (SSH) plutôt que sur un container Docker MySQL. Raison : le plan Docker supposait des fixtures .sql synthétiques pour alimenter le schéma Jeedom — le schéma Jeedom réel est trop complexe et divergent pour être maintenu fidèlement en fixture. La box réelle (ADR-0018) offre plus de valeur à coût inférieur.

**Mécanisme de skip** : les tests `tests/integration/` portent `@pytest.mark.integration` et skippent automatiquement si `/etc/holmes_mcp_ro.conf` est absent — zéro friction en CI, zéro friction en dev machine.

**CI GitHub Actions** (`ci.yml`) : lint `ruff` + tests unitaires uniquement, déclenchés sur push `develop` et PR vers `main`. Pas de job Docker MySQL.

**Couvertures cibles** :

- `_core/` et `_domain/` : >80% (unitaires + mocks)
- `_domain/sanitize.py` : **100% obligatoire**

**Deux jeux de fixtures** :

- **Synthétiques** : générées par Claude Code, versionnées dans `tests/fixtures/synthetic/`, utilisées en CI
- **Réelles** : récupérées via SSH par Claude Code, sanitisées, hors repo (workspace local uniquement)

**Aucune action opérationnelle PO** pour les tests unitaires et d'intégration. Validation Claude Desktop (D8.3 critère #4) = test ponctuel par release, conduit par le PO.

## Conséquences

- La CI publique ne dépend plus de Docker MySQL — plus simple, plus robuste
- Les tests d'intégration (SSH) valident les cas que les mocks ne peuvent pas couvrir : socket unix MariaDB, schéma Jeedom réel, logs réels
- L'auto-skip permet d'exécuter `pytest tests/` sans flag spécial hors box Jeedom
- Le niveau 100% sur `sanitize.py` est non négociable : filet de sécurité principal contre les fuites de credentials
