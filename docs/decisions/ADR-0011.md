# ADR-0011 — Licence AGPL-3.0 et bascule jeedom-skills MIT → AGPL

- **Date** : 2026-05-03 (esquisse) / TBD (finalisation J0)
- **Statut** : draft
- **Source brief** : D10.4

## Contexte

Holmes MCP est un plugin Jeedom natif open source. La licence doit être cohérente avec l'écosystème Jeedom (core AGPL-3.0), garantir que toute évolution dérivée reste accessible à la communauté, et s'aligner avec jeedom-skills (actuellement MIT). La bascule de jeedom-skills MIT → AGPL est actée conjointement à la sortie V1 Holmes MCP.

La condition préalable à la bascule de jeedom-skills est la vérification que le PO est l'unique copyright holder du repo (aucune contribution externe). Si contribution externe identifiée : escalade PO (consentement, réécriture, ou fallback MIT).

## Décision pressentie

**Holmes MCP** : licence **AGPL-3.0** dès la création du repo.

**jeedom-skills** : bascule **MIT → AGPL-3.0** annoncée conjointement à la sortie V1.0.0 Holmes MCP.

Conditions opérationnelles pour la bascule jeedom-skills :
1. Vérification J0 : `git log --pretty='%ae' | sort -u` côté jeedom-skills + confirmation PO = seul copyright holder
2. Si OK → Claude Code rédige l'ADR de relicence dans jeedom-skills, commit dédié `LICENSE` + entrée `CHANGELOG.md`
3. Communication forum Jeedom au moment de la sortie V1.0.0 Holmes MCP (annonce conjointe)

Cette ADR s'applique au repo Holmes MCP. L'ADR de relicence jeedom-skills est rédigée dans le repo jeedom-skills.

## Conséquences pressenties

- AGPL-3.0 garantit que tout fork ou service utilisant le code reste open source (si distribué)
- La bascule conjointe aligne la sphère jeedom-audit pour communication cohérente à la communauté
- Si le copyright holder n'est pas unique → option fallback : tout rester en MIT (décision PO)
