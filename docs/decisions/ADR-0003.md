# ADR-0003 — Version de la spec MCP cible V1

- **Date** : 2026-05-03 (esquisse) / 2026-05-03 (finalisation J1-1)
- **Statut** : accepted
- **Source brief** : D1.2

## Contexte

La spec MCP (Model Context Protocol, Anthropic) évolue avec des révisions datées (`2024-11-05`, `2025-03-26`, `2025-06-18`, etc.). Cibler une version précise et la figer dans une ADR est non négociable pour maintenir le plugin sans casser silencieusement des clients existants. La version cible détermine aussi les fonctionnalités disponibles — notamment le transport Streamable HTTP, introduit dans la révision 2025-03-26.

## Décision — J1-1 (D1.2 tranché)

**Spec MCP cible V1 : révision `2025-03-26` (Streamable HTTP)**

SDK Python retenu : `mcp==1.27.0` (PyPI, officiel Anthropic).

| Critère | Résultat |
|---|---|
| Dernière version stable au J1 effectif | `2025-03-26` ✅ |
| Compatibilité SDK MCP Python officiel | `mcp==1.27.0` ✅ |
| Support Streamable HTTP | ✅ (introduit en 2025-03-26) |
| Politique N et N-1 | N=2025-03-26, N-1=2024-11-05 |

**Transport retenu** : Streamable HTTP (`mcp.server.fastmcp.FastMCP`, méthode `streamable_http_app()` — confirmée à la ligne 950 du SDK sur la box PO).

**Version SDK épinglée** dans `plugin_info/packages.json` : `mcp==1.27.0`.

La version cible est documentée dans le README V1 pour que les utilisateurs vérifient la compatibilité de leur client MCP.

## Conséquences

- Breaking changes côté spec MCP → version majeure Holmes MCP (2.0.0) avec migration guide
- Support N et N-1 : quand une nouvelle révision sort, évaluation de la compatibilité + ADR d'adoption
- La version cible est la référence pour tous les tests d'intégration client (MCP Inspector, Claude Code)
- `mcp>=1.27.0,<2.0` dans `packages.json` pour permettre les patches sans forcer les upgrades majeurs
