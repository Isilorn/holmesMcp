# ADR-0003 — Version de la spec MCP cible V1

- **Date** : 2026-05-03 (esquisse) / TBD (finalisation J0)
- **Statut** : draft
- **Source brief** : D1.2

## Contexte

La spec MCP (Model Context Protocol, Anthropic) évolue avec des révisions datées (`2024-11-05`, `2025-03-26`, `2025-06-18`, etc.). Cibler une version précise et la figer dans une ADR est non négociable pour maintenir le plugin sans casser silencieusement des clients existants. La version cible détermine aussi les fonctionnalités disponibles — notamment le transport Streamable HTTP, introduit dans la révision 2025-03-26.

## Décision pressentie

**Délégué Claude Code J0** (🔵 D1.2) : cibler la **dernière version stable de la spec MCP au J0 effectif**, sous réserve de compatibilité avec le SDK MCP Python officiel retenu (ADR-0002).

Critères de sélection :
1. Dernière version stable au J0 effectif
2. Compatibilité confirmée avec le SDK MCP Python officiel
3. Support du transport Streamable HTTP (obligatoire — ADR-0001)
4. Politique de support N et N-1 (alignement jeedom-skills)

La version retenue sera documentée dans cette ADR au J0 avec : numéro de version précis, date de révision, URL de la spec, changelog notable. Elle sera mentionnée dans le README V1 pour que les utilisateurs puissent vérifier la compatibilité de leur client MCP.

## Conséquences pressenties

- Breaking changes côté spec MCP → version majeure Holmes MCP (2.0.0) avec migration guide
- Support N et N-1 : quand une nouvelle révision sort, évaluation de la compatibilité + ADR d'adoption
- La version cible est la référence pour tous les tests d'intégration client (MCP Inspector, Claude Desktop)
