# ADR-0016 — Observabilité

- **Date** : 2026-05-03 (esquisse) / TBD (finalisation J0)
- **Statut** : draft
- **Source brief** : §14 (Observabilité), D14.1-D14.9

## Contexte

Trois canaux de logs coexistent : (1) logs daemon Holmes MCP (sujet de cette ADR), (2) logs Jeedom natifs lus *par* Holmes MCP via tool `tail_log`, (3) logs côté client MCP (Claude Desktop — hors scope). Le daemon Python doit logguer suffisamment pour permettre au PO non-dev de diagnostiquer les problèmes sans ouvrir un terminal.

## Décision pressentie

**Format JSON Lines** pour les logs daemon (un objet JSON par ligne), schéma standardisé :

```json
{
  "ts": "2026-05-02T14:32:01.123Z",
  "level": "info",
  "request_id": "uuid-...",
  "user": "guillaume",
  "client": "claude-desktop/1.5",
  "tool": "list_scenarios",
  "params": { "active": true },
  "duration_ms": 45,
  "result": "ok",
  "rows_returned": 12,
  "filtered_fields": []
}
```

**Deux fichiers de log distincts** (D14.9) :
- `holmesMcp` — logs plugin PHP (install, démarrage/arrêt daemon, erreurs PHP)
- `holmesMcp_daemon` — logs daemon Python JSON Lines

**Niveau par défaut** : `info`. Configurable via la page de config plugin (dropdown : `debug` / `info` / `warning` / `error`). `debug` = traces SQL, HTTP, internes — verbose.

**Vue dédiée logs dans la page config** (D14.4) : tableau lisible parsant le JSON Lines. Filtres : par utilisateur, tool, niveau, fenêtre temporelle (dernière heure / jour / 7 jours). Refresh auto configurable (5s, 30s, off). Implémenté en J6 (🔵 D14.4 — framework JS et filtres exacts délégués J1).

**Ce qui n'est pas loggué** (D14.5) : contenu des résultats, contenu intégral des requêtes `query_sql` (résumé seulement), credentials, contenus `configuration` JSON même filtrés.

**Pas de télémétrie externe** (D14.7) : tous les logs restent locaux sur la box.

## Conséquences pressenties

- La vue UI logs permet au PO non-dev de diagnostiquer sans terminal
- JSON Lines facilite le parsing par la vue UI et par des outils externes si le PO le souhaite
- La sanitisation des logs est garantie (D14.5 + D15.4 : même sanitiseur que les réponses MCP)
