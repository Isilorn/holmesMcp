# ADR-0008 — Liste des 5 resources V1 minimales

- **Date** : 2026-05-03 (esquisse) / 2026-05-04 (accepted — J5-4)
- **Statut** : accepted
- **Source brief** : §6 (Resources MCP), D6.1-D6.5

## Contexte

Les resources MCP sont des URIs attachables par l'utilisateur dans son client (Claude Desktop : mécanisme "joindre une entité"). Elles permettent à l'utilisateur de pré-charger un contexte sans demander explicitement au LLM d'appeler un tool. En V1, l'objectif est minimal (effort marginal faible, bénéfice UX réel) : wrapping des tools existants, pas de schéma nouveau.

## Décision

**5 resources minimales V1**, chacune wrappant un ou des tools existants (pas de duplication de schéma — D6.4) :

| URI | Contenu | Tool(s) source |
|---|---|---|
| `jeedom://install/overview` | Snapshot général install | `get_install_overview` |
| `jeedom://install/health` | État de santé synthétique | `get_health_summary` |
| `jeedom://scenario/{id}` | Description complète scénario (structure + log) | `describe_scenario` + `get_scenario_log` |
| `jeedom://equipment/{id}` | Détail complet équipement | `get_equipment` |
| `jeedom://logs/today` | Agrégat logs core + scénarios des 24h | `tail_log` filtré |

**Approche d'énumération hybride** (D6.3) : `resources/list` retourne (a) les 3 resources globales statiques (overview, health, logs/today), (b) top-50 scénarios actifs + top-50 équipements actifs (énumération plafonnée, **plafond=50** décidé empiriquement — 62 scénarios et 217 éqLogics sur la box PO), (c) les 2 templates `jeedom://scenario/{scenario_id}` et `jeedom://equipment/{equipment_id}` retournés via `resources/templates/list`. **Résultat mesuré : 103 concrètes + 2 templates.**

**Note d'implémentation FastMCP** : les resources concrètes (URI sans `{var}`) doivent être enregistrées avec une fonction **sans paramètre**. Utilisé des factories `_make_scenario_fn(sid, ak)` / `_make_equipment_fn(eid, ak)` qui retournent des callables `() -> str`. Les resource templates (`{scenario_id}`, `{equipment_id}`) sont retournés séparément via `resources/templates/list` (pas dans `resources/list`).

**Stabilité semver** alignée sur les tools (D5.5) : les schémas des resources ne changent pas sur V1.x.

**Pas de prompts V1** (D6.5). Candidates roadmap V1.x.

## Conséquences

- Un seul point de maintenance : le schéma de chaque resource est le schéma du tool source
- Le plafond d'énumération est configurable et mesuré empiriquement sur la box du PO (J1)
- Les resources offrent une UX Claude Desktop améliorée sans surcoût de maintenance
