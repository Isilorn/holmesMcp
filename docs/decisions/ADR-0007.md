# ADR-0007 — Liste des 25 tools V1

- **Date** : 2026-05-03 (esquisse) / TBD (finalisation J0)
- **Statut** : draft
- **Source brief** : D5.3, D5.4, D5.5, D5.6, D5.6.bis, D5.7

## Contexte

Les tools MCP sont les verbes appelés par le LLM pour interroger la box. La liste V1 doit couvrir l'ensemble des besoins d'audit et de compréhension d'une installation Jeedom, tout en étant suffisamment granulaire pour que le LLM choisisse le bon tool. La stabilité des schémas in/out est critique (D5.5) pour la compatibilité avec jeedom-audit futur consommateur MCP.

## Décision pressentie

**25 tools en 6 familles** (liste figée en V1, sauf raffinements internes validés PO) :

**Famille 1 — Découverte d'install** : `get_install_overview`, `list_objects`, `list_plugins`, `get_config`

**Famille 2 — Équipements et commandes** : `list_equipments`, `find_equipments_advanced`, `get_equipment`, `find_equipment_by_name`, `list_commands`, `find_commands_advanced`, `get_command_history`

**Famille 3 — Scénarios** : `list_scenarios`, `find_scenarios_advanced`, `get_scenario`, `get_scenario_structure`, `describe_scenario`, `find_scenario_dependencies`, `get_scenario_log`

**Famille 4 — Variables / dataStore** : `list_datastore_variables`, `get_datastore_variable`

**Famille 5 — Logs et diagnostic** : `tail_log`, `list_log_files`, `get_health_summary`

**Famille 6 — Recherche transverse** : `search_text`

**Power-user / Audit** : `query_sql` (SELECT-only, blacklist tables sensibles, LIMIT obligatoire, sanitisation runtime, mini SQL cookbook dans la description)

**Politique semver des tools** (D5.5) :
- `1.0.x` : schémas inchangés
- `1.x.0` : nouveaux tools ou paramètres optionnels rétro-compatibles
- `2.0.0` : breaking changes possibles + migration guide

La liste a été raffinée en J1-2 via la matrice de couverture D5.8 (ADR-0019) : **0 ajout requis**. Les 25 tools couvrent intégralement les 13 WF jeedom-audit — liste figée pour V1.

## Conséquences pressenties

- Chaque tool a un module Python dédié dans `resources/holmesMcpd/tools/`
- Les descriptions des tools rappellent les conventions Jeedom pour guider le LLM
- `query_sql` inclus dès V1 préserve la flexibilité pour les workflows d'audit avancé et la future consommation par jeedom-audit
- La stabilité des schémas V1.x protège jeedom-audit qui pourra exiger `holmesMcp >= 1.0, < 2.0`
