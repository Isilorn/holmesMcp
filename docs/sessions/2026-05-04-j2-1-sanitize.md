## Livrables

### `resources/holmesMcpd/_domain/sanitize.py`

Module sanitisation complet, 3 mécanismes cumulatifs :

**Mech 1 — Whitelist par table** (`_TABLE_WHITELISTS`) :
- 13 tables couvertes : `eqLogic`, `cmd`, `scenario`, `object`, `dataStore`,
  `config`, `plugin`, `history`, `historyArch`, `scenarioElement`,
  `scenarioSubElement`, `scenarioExpression`, `interactDef`
- Tout champ absent de la whitelist → `FILTERED`
- Cas spécial `config` : key visible, value masquée si key sensible (transparence LLM)

**Mech 2 — Regex sur clés JSON** (`_BLOB_KEY_RE`) :
- Pattern D15.2 : `password|pwd|token|apikey|api_key|secret|hash|credentials|auth|cert|private_key|access_key|client_secret|bearer` (case-insensitive)
- Appliqué sur les colonnes blob : `configuration`, `options`
- Récursion sur les dicts imbriqués

**Mech 3 — Extras par plugin** (`_PLUGIN_EXTRA_KEYS`) :
- 15 plugins : jMQTT, Agenda, Alarme, Thermostat, Jeedom Connect, Script, MQTT Manager,
  Virtuel, Aqara, Zigbee2MQTT, Sonos, Philips Hue, Z-Wave, Netatmo, ecodevices, rfxcom
- Nommages exotiques non couverts par la regex : `pin`, `code`, `armCode` (Alarme),
  `installCode` (Jeedom Connect), `login`/`user`/`username` (MQTT Manager, jMQTT...), etc.

**API publique** :
- `is_sensitive_key(key, plugin=None) → bool`
- `sanitize_json_blob(blob, plugin=None) → (dict|str|None, list[str])`
- `sanitize_row(row, table=None, plugin=None) → (dict, list[str])`
- `sanitize_rows(rows, table=None, plugin=None) → (list[dict], list[str])`
- `wrap_result(data, filtered_fields) → dict`
- Constante `FILTERED = "***FILTERED***"`

### `tests/unit/_domain/test_sanitize.py`

158 tests — **100% couverture** (82/82 statements).

Couverture des cas :
- `is_sensitive_key` : 36 patterns regex, 30+ non-sensibles, tous les extras plugin
- `sanitize_json_blob` : None, vide, JSON invalide, array, scalar, dict, nested, plugin extras
- `_sanitize_config_row` : key sensible/non-sensible, key absente/vide
- `sanitize_row` : toutes les 13 tables (whitelist), blob columns, regex défense en profondeur, table config
- `sanitize_rows` : liste vide, déduplication, tri
- `wrap_result` : list vs dict data
- 10 fixtures "no credential in output" (D15.5) — chaque fixture vérifie qu'aucun
  credential connu ne traverse la sanitisation
