# Session J0-1b — Scripts dev sudo temporaire (2026-05-03)

**Jalon** : J0 (suite J0-1)
**Type** : Outillage dev — pas de SSH sur box Jeedom

---

## Objectif de la session

Adapter les scripts fleet `add-sudo-temp.sh` et `remove-sudo-temp.sh` pour cibler une seule machine (alias SSH) et intégrer la lecture du mot de passe depuis `secrets.cfg` sans affichage en clair.

---

## Artefacts produits / modifiés

| Fichier | Changement |
|---|---|
| `dev/add-sudo-temp.sh` | Réécriture : `$1`=alias SSH, `$2`=secrets.cfg, password via `set +x` + `unset`, sans `fleet-ssh-config` |
| `dev/remove-sudo-temp.sh` | Réécriture : `$1`=alias SSH, sans password (NOPASSWD actif), sans `fleet-ssh-config` |
| `dev/secrets.cfg` | Nettoyage : suppression de tout sauf `SUDO_PASS` (Gmail, clés SSH, Git, réseau, ROLES supprimés) |

---

## Résultat des tests

```
$ ./dev/add-sudo-temp.sh Jeedom dev/secrets.cfg
Machine                      Statut
-------                      ------
Jeedom                       OK

$ ./dev/remove-sudo-temp.sh Jeedom
Jeedom                       SUPPRIMÉ

$ ./dev/remove-sudo-temp.sh Jeedom   ← idempotence
Jeedom                       DÉJÀ ABSENT
```

---

## Décisions techniques

- `fleet-ssh-config` supprimé : alias `Jeedom` déjà dans `~/.ssh/config`
- Password jamais affiché : `set +x` avant sourcing, `unset SUDO_PASS` + `unset PASS_B64` après usage
- `remove` sans secrets.cfg : NOPASSWD est actif au moment de la suppression, pas besoin du mot de passe
